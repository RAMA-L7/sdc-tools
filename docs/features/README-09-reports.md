# Feature 9: HTML Signoff Reports

> **Module:** `reporter.py` · **CLI:** `sdc-tools report`

---

## Why It's Needed

Terminal output and Streamlit UIs are great for interactive work, but signoff requires **static, portable, self-contained reports** that can be:
- Emailed to stakeholders
- Archived in a revision control system
- Printed or attached to design reviews
- Shared with teams who don't have the tools installed

The HTML report generator produces **zero-dependency, self-contained HTML files** — all CSS is inline, no external stylesheets, no JavaScript, no CDN references. Every report is a single `.html` file that renders beautifully in any browser.

---

## How It Was Implemented

### Architecture

- **Inline CSS template** — 100+ lines of embedded styles in `_CSS` string
- **`_page()` base template** — wraps any body content with standard header, title, subtitle, and footer (auto-generated date + version)
- **Reusable components** — `_summary_metrics()`, `_badge()`, `_issue_rows()`, `_table()`
- **5 report generators** — check, diff, clock-relations, rules, coverage
- **Coverage-specific CSS** — `_COV_CSS` (40 lines) for progress bars and score displays
- **Rules Registry integration** — each issue row looks up the rule name for the tooltip column

### Available Reports

| Report Type | Function | CLI Command | Content |
|-------------|----------|-------------|---------|
| **SDC Quality** | `generate_check_report()` | `report check` | Issues table, stats grid, severity badges |
| **Change Impact** | `generate_diff_report()` | `report diff` | Change cards, metrics, V1/V2 counts |
| **Clock Relations** | `generate_clock_report()` | `report clock-relations` | Mismatch cards, clock definitions, N×N matrix |
| **Rules Registry** | `generate_rules_report()` | (API only) | Rule table with code, severity, module, name |
| **Coverage** | `generate_coverage_report()` | `report coverage` | Score display, category progress bars, missing items |

## Use Cases

| Scenario | Report Type | Why |
|----------|-------------|-----|
| **Signoff meeting** | SDC Quality Report | Print and review as a team |
| **ECO approval** | Change Impact Report | Document what changed between revisions |
| **Clock review** | Clock Relations Report | Share clock domain analysis with CDC engineers |
| **Constraint completeness audit** | Coverage Report | Demonstrate signoff readiness with metrics |
| **Project archive** | All types | Attach to milestone documentation |

## CLI Usage

```bash
# SDC Quality Report
sdc-tools report check design.sdc --output quality_report.html

# With info-level items included
sdc-tools report check design.sdc --verbose -o report.html

# Change Impact Report
sdc-tools report diff old.sdc new.sdc \
  --v1-name "RTL Freeze" \
  --v2-name "ECO #42" \
  --output diff_report.html

# With linked TCL files
sdc-tools report diff old.sdc new.sdc \
  --linked-v1 params_v1.tcl \
  --linked-v2 params_v2.tcl \
  -o diff_report.html

# Clock Relations Report
sdc-tools report clock-relations design.sdc -o clock_report.html

# Coverage Report
sdc-tools report coverage design.sdc -o coverage_report.html
```

## Python API

```python
from reporter import (
    generate_check_report,
    generate_diff_report,
    generate_clock_report,
    generate_coverage_report,
    generate_rules_report,
)
from checker import check_sdc
from rules_registry import get_all_rules

# Check report
with open("design.sdc") as f:
    result = check_sdc(f.read())
html = generate_check_report(result, "design.sdc", verbose=True)
with open("report.html", "w") as f:
    f.write(html)

# Coverage report
from coverage import parse_sdc_coverage
cov_result = parse_sdc_coverage(sdc_text, "design.sdc")
cov_html = generate_coverage_report(cov_result, "design.sdc")

# Rules registry report
rules_html = generate_rules_report(get_all_rules(), "All Rules")
```
